# Project Objective: SOCKS5-VPS toolkit — доступ к Claude / Google Antigravity из РФ

Набор скриптов + гайд, поднимающие прокси-цепочку, где трафик выходит в интернет
с иностранного (или резидентного) IP. Цель — рабочий, проверенный, безопасный
инструмент для не-администратора. Ветка: `add-socks5-vps-toolkit` (baseline =
пустой `main`; `git diff main...HEAD` = весь набор).

## 🧠 Dev Notes & Hypotheses (Bugs, ideas, context)

- **Корень повторяющихся багов — дивергенция 5 копий** общих функций
  (валидаторы / детект SSH-портов / ufw-гейт / оркестрация). Устранено
  структурно: всё в `scripts/_lib.sh`. Любую общую правку делать ТОЛЬКО там.
- **Два анти-локаут-ядра проверены СИМУЛЯЦИЕЙ** (не «на глаз»):
  `ufw_orchestrate` (re-run / first-run-ok / 0-SSH-allow→не-enable / enable-fail)
  и `ufw_clear_port` (SSH-порты не трогает). Инвариант «не включить ufw без
  живого SSH-allow» держится.
- **Trade-off принят:** ради устранения дрейфа скрипты больше НЕ single-file —
  `_lib.sh` нужен рядом, запуск только файлом (не `bash -s < …`). Это в README/гайде.
- **Грабли инструментария (для будущих правок через агентов):** при emit-е
  содержимого из JSON-результата воркфлоу агент иногда over-экранирует кавычки
  (`\"`) → файл ломается. ВСЕГДА после записи: `bash -n` + grep `\\"`. `_lib.sh`
  так уже один раз ловили и восстанавливали.
- **Модель угроз:** оператор сам задает секреты, root на своем VPS; многие
  «инъекции» требуют уже иметь root → они теоретические. Реально важны
  анти-локаут, экспозиция периметра, приватность логов на РФ-узле, цепочка поставки.
- **Цикл review→fix осознанно остановлен** (≈5 кругов): exhaustive recall на
  большом bash-наборе бездонен, маржинальная безопасность круга ~0, правки
  иногда заносят новый edge. Дальше — реальный прогон, а не N+1 синтетический ревью.
- `bash` локально = Git Bash (Windows); `ufw`/`detect_ssh_ports` на Windows
  пусты — поэтому ядра тестировались моками/симуляцией, не на живом ufw.
